### ERROR 2: APScheduler / Scheduled Jobs DO NOT EXIST
The report (Section 20.2) listed 7 scheduled cron jobs:
- Leaderboard cache refresh (every hour)
- Expired notification cleanup (daily)
- Daily quiz generation (daily)
- Inactive user reminder (weekly)
- License expiry check (daily)
- Analytics aggregation (daily)
- Monthly credit reset

**Reality from `app/lifespan.py`:**  
The entire `lifespan()` function contains:
1. Redis connection setup
2. Thread pool executor init
3. Static directory creation
4. MongoDB index creation
5. Sample data initialization

**There is NO APScheduler, NO cron jobs, NO scheduled tasks.**  
The "daily quiz" is seeded from a date-based `random.seed()` on each request (see `daily_quiz.py` line 104), not generated by a scheduler.

**→ Entire Section 20.2 is FABRICATED. Mark all 7 items ❌.**

### ERROR 5: XP Level Formula is Different
The report stated (Section 6.3):
```
level = floor(XP / 100) + 1
```

**Reality from `app/utils/helpers.py` (lines 385–400):**
```python
async def calculate_level(xp: int) -> dict:
    level = 1
    xp_for_next = 1000  # Starts at 1000 XP, NOT 100
    while xp >= xp_for_next:
        xp -= xp_for_next
        level += 1
        xp_for_next = int(xp_for_next * 1.5)  # Increases by 1.5x each level
```
The actual formula is a **progressive XP ladder** (1000 → 1500 → 2250 → ...), not a simple `floor(XP/100)+1`.

**→ Section 6.3 XP formula is WRONG. Correct it.**

### MISSING 12: Compiler Has Docker FALLBACK to Local Execution
The report described Docker as the only execution mode. In reality:
- **Docker is checked on startup** (`is_docker_available()` cached)
- **If Docker is unavailable**, falls back to local subprocess execution using system-installed runtimes
- `execution_mode` field in response tells the UI which mode was used (`"docker"` or `"local"`)
**→ This fallback mechanism is architecturally important for migration and was not documented.**
